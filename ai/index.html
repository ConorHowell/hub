Introduction

A pickup-and-play resource for Claude Code — skills, bot templates, and setup guides for AI-assisted development.

Claude Code is Anthropic's CLI for Claude — a terminal-based AI assistant for software engineering tasks. It reads your project files, runs commands, and edits code directly.

Install from claude.ai/code or via the VS Code extension (search "Claude Code" by Anthropic).

This hub provides everything you need to go from zero to a fully configured AI development setup: global rules, slash command skills, and drop-in bot templates (CLAUDE.md files).

Quick Start

Ideal order for a new machine or new project — step 1 to fully configured bot.

1
Install Claude Code
VS Code: Extensions → search "Claude Code" by Anthropic. CLI: claude.ai/code
2
Set global rules — ~/.claude/CLAUDE.md
Download a global rules template from this hub. Drop it at ~/.claude/CLAUDE.md. Global rules apply to every project and every session — put session-start behavior and safety rules here.
3
Install Caveman first — token savings from session 1
Skills install globally — run once from anywhere, active in all projects.
npx skills add <path-to-caveman-skill-dir> -a claude-code -y
4
Install remaining skills
Repeat for each skill, or run the setup.sh loop to install all at once.
npx skills add <path-to-skill-dir> -a claude-code -y
5
Add a project CLAUDE.md
Drop a CLAUDE.md at your project root. Claude reads it at every session start. Project rules override global rules. Use a template from the Bot Templates section.
6
Design specialized prompts with /prompt-engineer
For any specialized agent role — invoke and answer the 6 intake questions. Works for any domain.
7
Verify everything works
Start a Claude Code session. Type /caveman — a terse response confirms skills are loaded. Installed skills also appear listed in the session startup context.

Global Rules

The ~/.claude/CLAUDE.md file sets Claude's baseline behavior across every project and session.

Two scopes of CLAUDE.md files exist:

  • ~/.claude/CLAUDE.md — global rules, apply across all projects and sessions
  • <project>/CLAUDE.md — project-specific rules, override globals

A good CLAUDE.md is short and scannable. Include:

  • Role and persona for this project
  • Project structure and key files
  • Rules — what to always do, what to never do
  • Next Steps — current in-progress work so any bot can pick up immediately

Skills — Slash Commands

Skills extend Claude Code with slash commands. Install once globally, active in every project and session.

Global Install

Skills install into ~/.claude/ — run once from anywhere, active in all projects. The -a claude-code flag targets Claude Code. -y skips confirmation.

npx skills add <path-to-skill-directory> -a claude-code -y

Common failure: the directory must contain a SKILL.md file with valid name: and description: frontmatter. Missing file or malformed frontmatter = silent failure.

Write Your Own Skill

Subagent / Worktree Safety Rules

When Claude spawns agents that use git worktrees, follow these rules to prevent silent failures.

Use absolute paths with -C — never cd

The working directory persists across bash calls. cd inside one call affects all subsequent calls in the session, causing git commands to target the wrong repo silently.

# CORRECT — no CWD drift between bash calls git -C /absolute/path/to/repo status git -C /absolute/path/to/repo log --oneline -3 # WRONG — cd persists; later git commands run against wrong directory cd /some/path && git status

Identify all git repos before delegating

Subdirectories can have their own .git and are separate repos. Check:

ls <subdir>/.git # exists = separate repo

Tell the agent explicitly which git repo each file belongs to when multiple repos exist in the tree.

After a worktree agent completes — merge explicitly

Worktree agents commit to a temp branch, not main. Never assume the merge happened automatically.

git -C /abs/path log --oneline worktree-branch -3 # verify commit is there git -C /abs/path merge worktree-branch --ff-only # fast-forward into main git -C /abs/path log --oneline -3 # confirm main now has it ls /abs/path/new-file # verify file is on disk

Never assume "already up to date" means success — it may mean the merge ran against the wrong directory.

Pre-ship Review Pattern

Run a review skill automatically before committing to catch bugs before they ship.

REVIEW=$(echo "$DIFF" | claude --print "$REVIEW_PROMPT") VERDICT=$(echo "$REVIEW" | head -1 | tr '[:lower:]' '[:upper:]') [ "$VERDICT" = "APPROVE" ] || read -p "Issues flagged. Ship anyway? (y/N): " CONFIRM

The /ship-review skill provides a ready-made review prompt for this pattern. Wire it into your deploy or commit script.
